# LumenMark v0.3.11 回归测试报告

## 测试范围

- 自动更新：菜单入口、更新状态弹窗、下载进度、安装完成后重启入口、签名 updater 配置。
- 主题系统：内置主题、官方主题、导入主题、设置弹窗、主题菜单、应用壳变量。
- 可读性：工作区、大纲、搜索框、状态栏、链接弹框、代码块、Mermaid journey、图片/图表画布。
- 回归：导入 CSS 安全处理、Mermaid/LaTeX 预览、文档编辑区布局。

## 自动化测试

- `npm run test`：覆盖官方主题注册、主题对比度、更新弹窗、Release updater 配置、应用菜单命令、工作区与大纲主题变量。
- `cargo test --manifest-path src-tauri/Cargo.toml`：覆盖菜单命令、设置与检查更新菜单位置、主题菜单映射、工作区安全逻辑。
- `npm run build`：覆盖 TypeScript 与生产构建。
- `git diff --check`：覆盖空白和补丁格式。

## 主题可读性 QA

使用 `docs/qa/v0.3.11-theme-sample.md` 逐项检查：

- GitHub Light：正文、侧栏、链接、表格、代码块、Mermaid、图片可读。
- GitHub Dark Dimmed：Mermaid journey 和图片使用浅色高对比画布，标线可见。
- GitHub Dark Colorblind：节点、连线、代码 token 和链接具有足够区分度。
- Oh My Zsh Dark：终端风格下侧栏、大纲、搜索框文字保持可读。
- Monokai Terminal：高饱和 token 不影响正文和弹框可读性。
- Solarized Light：低对比浅色风格仍保持正文、代码和边框清晰。
- Lumen Paper / Lumen Ink：长文写作场景下正文、代码、公式和图表稳定可读。

## 更新流程 QA

- GitHub Secrets 已配置 `TAURI_SIGNING_PRIVATE_KEY` 和 `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`。
- 仓库只提交 updater public key，不提交私钥。
- Release workflow 使用签名 secret 生成 updater artifacts。
- `latest.json` endpoint 配置为 GitHub Release latest 下载地址。

## 已知风险

- Windows 安装包仍未代码签名，首次运行可能出现 SmartScreen 提示。
- macOS 仍为 ad-hoc 签名，尚未 Apple 公证。
- 自动更新从 v0.3.11 开始对后续版本生效；已经安装的 v0.3.10 不包含 updater 客户端，无法通过应用内自动升级到 v0.3.11。
- 导入主题仍只支持单个 CSS 文件，不支持字体、图片和完整主题文件夹 assets。

## 敏感信息检查

发布前需执行密钥、证书、`.env`、私有 URL、本地个人路径和账号信息扫描。Updater 私钥只存在 GitHub Secrets，不写入仓库。
